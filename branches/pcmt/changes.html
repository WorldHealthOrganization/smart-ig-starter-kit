Release (V2.1.0)

L2 DAK Templates

L2 DAK templates were updated as described in the table below:

Template Changes

The DAK document (the main template)

·    Copyright text was added on the "Cover" tab.

·    The column "Activity ID and name" was split in 2 independent columns "Activity ID" and "Activity name".

Core data dictionary

·  Copyright text was added on the "Cover" tab.

·  The column "Activity ID and name" was split in 2 independent columns "Activity ID" and "Activity name".

Decision-support logic

·  Copyright text was added on the "Cover" tab.

·  Decision-support table ID was suffixed with “Sequential number of the decision-support table” to allow linkage of multiple tables to one activity:
    Old notation example: IMMZ.D2.DT
    New notation example: IMMZ.D2.DT.1

·  The notation for the “Rule ID” was simplified:
    Old notation example: IMMZ.D2.DT.01
    New notation example: 1

·  The column "Activity ID and name" was split in 2 independent columns "Activity ID" and "Activity name".

Scheduling logic

·  Copyright text was added on the "Cover" tab.

·  Scheduling table ID was suffixed with “Sequential number of the scheduling logic table” to allow linkage of multiple tables to one activity:
    Old notation example: IMMZ.D2.S
    New notation example: IMMZ.D2.S.1

·  The notation for the “Rule ID” was simplified:
    Old notation example: IMMZ.D2.S.01
    New notation example: 1

·  The column "Activity ID and name" was split in 2 independent columns "Activity ID" and "Activity name".

Indicators and performance metrics

·  Copyright text was added on the "Cover" tab.

High-level functional and non-functional requirements

·  Copyright text was added on the "Cover" tab.

·  The column "Activity ID and name" was split in 2 independent columns "Activity ID" and "Activity name".

License page

The license details were updated and a new table "Scenarios for usage of WHO SMART Guidelines products" was added with the purpose of presenting WHO position in different scenarios of usage for WHO SMART Guidelines products.

Authoring a L2 DAK

The diagram presenting the relationship between DAK components was updated.

  • Updated IG publication documentation and some changes in authoring conventions
  • New page around SMART Guidelines COP added
